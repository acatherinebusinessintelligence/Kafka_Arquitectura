Kafka + Podman-Compose

1) Arranque limpio del cluster (3 brokers)

podman-compose -f podman-compose-kafka-3B.yml -p kafka-lab3 up -d

2) Checks esenciales

Contenedores activos

podman ps --format "{{.Names}}	{{.Status}}"

Brokers en Zookeeper

podman exec -it kafka1 bash -lc 'zookeeper-shell zookeeper:2181 ls /brokers/ids'

3) Crear, describir y probar tópicos

podman exec -it kafka1 bash -lc   'kafka-topics --create --topic demo    --bootstrap-server kafka1:29092    --partitions 3 --replication-factor 3 --if-not-exists'

4) Prueba de tolerancia a fallos

Ejemplo: parar un broker y seguir produciendo/consumiendo mensajes.

podman stop kafka3
podman start kafka3

5) Problemas comunes & soluciones

Contenedor ya existe 
podman rm -f kafka1 kafka2 kafka3 zookeeper
Solo aparece 1 broker

Usaste un YAML con un solo servicio. Asegúrate de ejecutar el archivo con kafka1, kafka2, kafka3.

Error de replicación RF=3

Revisa que Zookeeper muestre [1,2,3] y que los 3 contenedores estén arriba.

Consumer TimeoutException

Ocurre si no hay mensajes nuevos y se usa --timeout-ms. Es normal: usa --max-messages o quita el timeout.
